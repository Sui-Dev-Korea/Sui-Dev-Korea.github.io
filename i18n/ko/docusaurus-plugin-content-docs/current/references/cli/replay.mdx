---

Replay는 특정 transaction을 replay하여 transaction 효과를 확인하고 분석할 수 있는 도구이다. 간단히 말해, replay 도구는 transaction을 로컬에서 실행하고 해당 효과를 체인에 존재하는 효과와 비교한다.

Replay 도구를 사용하면 `--trace` 플래그를 전달하여 Move 디버거에서 사용할 수 있는 transaction 실행 추적을 생성할 수 있다. Move 디버거에 대한 자세한 내용은 [Move Debugger](../ide/debugger.mdx) 페이지를 참조한다. [gas profiling](./trace-analysis.mdx)과 같은 것은 다양한 [다양한 트랜잭션 추적을 분석](./trace-analysis.mdx#profiling-a-transaction) 할 수 있다

## Usage

Transaction을 재생하는 주요 명령은 다음과 같다. 사용 가능한 모든 플래그와 인수를 보려면 `sui client replay-transaction --help` 실행한다.

```
sui client replay-transaction --tx-digest <digest>
```

해당 명령을 실행하면 도구는 현재 디렉터리에 digest에 해당하는 디렉터리를 생성한다. `<cur_directory>/.replay/<digest>` , 단, `--output-dir` 플래그를 설정하지 않은 경우이다. 이전 명령을 성공적으로 실행하면 콘솔에 다음과 유사한 응답이 표시된다:
